# task 4
## ls.c
### void parse_args
Парсинг аргументов командной строки.

    Инициализирует структуру Flags нулевыми значениями
    Выделяет память для массива targets (целей - файлов/каталогов)
    Проходит по всем аргументам argv (кроме argv[0], который содержит имя программы)
    Определяет тип каждого аргумента:
        Если начинается с - (например, -l, -R):
            Обрабатывает каждый символ флага
            Устанавливает соответствующие поля в структуре Flags
        Иначе (например, file.txt, /home/user):
            Добавляет аргумент в массив targets
    Если нет целей: добавляет текущий каталог (.) как цель по умолчанию

### void get_mode_string
Преобразование битовой маски прав доступа в строку.

    Принимает mode_t mode (битовую маску прав доступа из struct stat)
    Заполняет строку str форматом типа: drwxr-xr--
        Позиция 0: Тип файла:
            d - каталог
            c - символьное устройство
            b - блочное устройство
            p - именованный канал (FIFO)
            l - символическая ссылка
            s - сокет
            - - обычный файл
        Позиции 1-3: Права владельца (user): rwx
        Позиции 4-6: Права группы (group): rwx
        Позиции 7-9: Права остальных (other): rwx

### void process_symlink
Обработка символических ссылок для красивого вывода.

    Читает цель символической ссылки с помощью readlink()
    Проверяет, является ли цель также символической ссылкой
    Выводит информацию в формате:
        Обычная ссылка: file_link -> real_file
        Цепочка ссылок: link1 -> link2 -> final_file

### void print_long_format
Форматированный вывод подробной информации о файле (аналог ls -l).

    Получает строку прав доступа через get_mode_string()
    Получает информацию о владельце (getpwuid()) и группе (getgrgid())
    Форматирует время модификации
    Выводит строку в формате:
      [права] [ссылки] [владелец] [группа] [размер] [дата] [имя] [-> цель_ссылки]
    Учитывает флаг -g:
        С флагом -g: показывает только группу
        Без флага -g: показывает владельца и группу

### void process_entry
Обработка одного элемента (файла или подкаталога).

    Формирует полный путь к файлу/каталогу
    Получает информацию о файле через lstat() (работает и со ссылками)
    Вызывает соответствующую функцию вывода:
        Если flags.long_format = 1: вызывает print_long_format()
        Иначе: проверяет тип файла:
            Для символических ссылок вызывает process_symlink()
            Для остальных просто выводит имя

### void process_directory
Обработка содержимого каталога.

    Открывает каталог с помощью opendir()
    Если это рекурсивный вызов (recursive_call = 1): выводит заголовок с путем к каталогу
    Читает все записи в каталоге с помощью readdir()
    Для каждой записи:
        Пропускает . и ..
        Пропускает скрытые файлы (начинающиеся с .), если не используется -l
        Вызывает process_entry() для обработки
    Закрывает каталог
    Если установлен флаг -R: повторно открывает каталог и рекурсивно обрабатывает все подкаталоги
